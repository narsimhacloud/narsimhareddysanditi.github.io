3. LDAP Domain joining

Project Description

This project focuses on automating the process of joining Linux servers to an LDAP domain using Ansible. The automation simplifies and streamlines what is typically a manual and time-consuming process, ensuring consistency and reducing configuration errors.

Project Overview:

Automated Domain Join:

Leveraged Ansible playbooks to automatically configure and join Linux servers to an LDAP domain.

Configuration Management:

Handled the setup of necessary configurations such as krb5.conf, sssd.conf, and nsswitch.conf to enable domain communication.

Customizable Variables:

Implemented flexible playbooks using parameterized variables for hostnames, Organizational Units (OUs), domain controllers, and credentials, making the solution reusable across environments.

Security Integration:

Integrated secure password management and authentication through Kerberos, ensuring robust and secure connections to the LDAP domain.

Post-Join Configuration:

Automated additional tasks such as configuring DNS settings, opening required ports, and performing security hardening after the domain join.

4. Automate the post Build configurations in RHEL-9 Servers

Project Description

RHEL 9 Post-Build Configuration Automation. This project focuses on automating post-build configurations for Red Hat Enterprise Linux (RHEL) 9 environments. The automation ensures that newly provisioned RHEL 9 systems are configured according to organizational standards, optimized for performance, and hardened for security.

Key Tasks:

Disk Formatting and Mounting:

  • Automated the formatting of new disks using filesystems like XFS or ext4.
  • Configured the mounting of disks to the correct paths (e.g., /var, /tmp, /home, etc.) with persistence across reboots.
  • Ensured swap partitions are properly set up with customizable sizes.

    • Network Configuration:

  • Configured network interfaces, setting up static IPs, hostnames, DNS, and routing based on the environment requirements.
  • Automated the setup of firewall rules using firewalld, ensuring that only necessary ports are open.

    • Security Hardening:

  • Implemented best practices for security hardening, including disabling unnecessary services, ensuring strong password policies, and configuring SELinux.
  • Applied SSH and sudo configurations to ensure secure access to the system, including enabling key-based authentication.

    • Nexpose Scans & Vulnerability Management:

    Integrated Nexpose or similar vulnerability management tools to perform automated security scans after system build, reporting vulnerabilities for further action.

    User Account Management:

  • Automated the creation of user accounts, including setting up appropriate groups, permissions, and home directories.
  • Configured sudo access for specific users, ensuring least privilege is applied.

    • Performance Tuning:

  • Applied performance optimizations for system resources such as CPU, memory, and I/O based on system requirements.
  • Tuned kernel parameters (sysctl settings) to improve system stability and performance in production environments.

  • Package Installation and Updates:

  • Automated the installation of essential packages and dependencies, such as security updates, development tools, and network utilities.
  • Configured the system to check for and apply automatic updates where necessary, ensuring security patches are always applied.
